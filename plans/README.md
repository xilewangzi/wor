## 🎯 核心功能概览

### 需求1：多端支付订阅
- **iOS**: Apple IAP（基于现有 sn-uts-iap 模块）
- **Android**: 小米、VIVO、荣耀三个厂商支付
- **鸿蒙**: 华为IAP

### 需求2：视频引导页
- 首次启动展示视频引导
- 支持跳过功能
- 本地存储引导状态

### 需求3：多渠道打包
- 5个渠道包：iOS、小米、VIVO、荣耀、鸿蒙
- 自动化打包脚本
- 渠道配置管理

## 🏗️ 技术架构亮点

### 1. 统一支付接口层
```
IPaymentService (统一接口)
    ├── IOSPaymentService
    ├── XiaomiPaymentService
    ├── VivoPaymentService
    ├── HonorPaymentService
    └── HarmonyPaymentService
```

### 2. 工厂模式
根据平台和渠道自动创建对应的支付实现

### 3. 状态管理
使用 Pinia 管理订阅状态、用户状态、支付状态

### 4. 条件编译
使用 uni-app 条件编译区分不同平台代码

## 💡 技术难点与解决方案

### 难点1：多端支付SDK集成
**解决方案**: 使用UTS插件封装原生SDK + 统一接口层

### 难点2：订阅状态同步
**解决方案**: 本地缓存 + 服务器验证双重机制

### 难点3：视频资源优化
**解决方案**: 压缩视频（<5MB）+ H.264编码

### 难点4：渠道包管理
**解决方案**: 配置文件 + 自动化打包脚本
